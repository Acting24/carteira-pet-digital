### [SLIDE 3 - PROJETO ESCOLHIDO]

**"Agora vou falar sobre o projeto que escolhi: Carteira Digital de Vacinação para PET.**

**O PROBLEMA:**
Atualmente, os donos de animais de estimação utilizam carteiras de vacinação em papel, fornecidas pelas clínicas veterinárias. O problema é que:
- Essas carteiras se perdem facilmente
- Não há sistema de alertas para próximas vacinas
- É difícil organizar o histórico completo, especialmente para quem tem múltiplos pets
- As informações ficam vulneráveis a danos (água, rasgos, etc.)

**O PÚBLICO-ALVO:**
Donos de animais de estimação - cães, gatos e outros pets. São pessoas de qualquer faixa etária que precisam gerenciar a saúde de seus animais.

**A SOLUÇÃO PROPOSTA:**
Um sistema web que permite:
- Cadastrar múltiplos pets
- Registrar histórico completo de vacinação
- Receber alertas automáticos sobre vacinas próximas do vencimento
- Acessar de qualquer lugar (computador, tablet, celular)
- Manter as informações seguras e organizadas

**POR QUE É RELEVANTE?**
Segundo dados do IBGE, existem mais de 140 milhões de animais de estimação no Brasil. A saúde preventiva, incluindo vacinação, é fundamental para o bem-estar animal e também para a saúde pública, evitando doenças como raiva e leptospirose."

### [SLIDE 4 - METODOLOGIA: DESIGN THINKING]

**"A disciplina nos orienta a usar a metodologia DESIGN THINKING. Vou explicar rapidamente como ela funciona e em que fase estamos.**

O Design Thinking é composto por **5 fases**:

**FASE 1: EMPATIA** 👈 *Estamos aqui!*
- **Objetivo:** Entender profundamente o usuário
- **O que fazemos:**
  - Definir quem é o usuário (no meu caso: donos de pets)
  - Entrevistar pessoas que têm pets
  - Mapear suas dores e necessidades
  - Observar como controlam as vacinas hoje
- **Duração:** Quinzenas 1-2

**FASE 2: DEFINIÇÃO**
- **Objetivo:** Sintetizar o problema
- **O que faremos:**
  - Analisar os dados das entrevistas
  - Criar uma "persona" (usuário típico)
  - Escrever uma declaração clara do problema
  - Priorizar necessidades
- **Duração:** Quinzenas 2-3

**FASE 3: IDEAÇÃO**
- **Objetivo:** Gerar ideias de solução
- **O que faremos:**
  - Brainstorming de funcionalidades
  - Desenhar wireframes (esboços de telas)
  - Avaliar viabilidade técnica
  - Selecionar as melhores ideias
- **Duração:** Quinzenas 4-5

**FASE 4: PROTOTIPAÇÃO**
- **Objetivo:** Construir a solução
- **O que faremos:**
  - Desenvolver o MVP (Produto Mínimo Viável)
  - Implementar funcionalidades essenciais
  - Criar interface funcional
- **Duração:** Quinzenas 6-10

**FASE 5: TESTE**
- **Objetivo:** Validar e melhorar
- **O que faremos:**
  - Testes com usuários reais
  - Coletar feedback
  - Corrigir bugs e problemas
  - Documentar aprendizados
- **Duração:** Quinzenas 11-14

Como podem ver, estamos na **Fase 1 - EMPATIA**, focando em entender profundamente as necessidades dos donos de pets."
