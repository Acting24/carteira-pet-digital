## ✅ **Comportamentos Esperados**

### **💻 No Desenvolvimento:**
- Seguir convenções de código estabelecidas pela equipe
- Fazer commits regulares com mensagens descritivas
- Testar código antes de fazer push
- Revisar Pull Requests com atenção e construtividade
- Documentar funcionalidades implementadas

### **💬 Na Comunicação:**
- Usar linguagem respeitosa no WhatsApp e GitHub
- Ser claro e objetivo nas mensagens
- Responder mensagens importantes em tempo hábil
- Pedir ajuda quando necessário sem receio
- Oferecer ajuda quando possível

### **👥 No Trabalho em Equipe:**
- Participar ativamente das discussões técnicas
- Respeitar decisões tomadas coletivamente
- Comunicar mudanças que possam afetar outros integrantes
- Ser flexível e adaptável às necessidades do projeto
- Celebrar sucessos da equipe

### **🎯 Na Resolução de Problemas:**
- Abordar conflitos de forma madura e profissional
- Focar na solução, não no problema
- Buscar consenso antes de tomar decisões importantes
- Escalar problemas para o líder quando necessário
- Manter transparência sobre dificuldades

---

## ❌ **Comportamentos Inaceitáveis**

### **🚫 Absolutamente Proibidos:**
- Linguagem ofensiva, discriminatória ou inadequada
- Ataques pessoais ou críticas destrutivas
- Assédio de qualquer tipo
- Compartilhamento não autorizado de informações do projeto
- Sabotagem do trabalho de outros integrantes

### **⚠️ Comportamentos Desencorajados:**
- Commits diretos na branch `main` sem aprovação
- Ignorar convenções estabelecidas pela equipe
- Não comunicar ausências ou atrasos
- Deixar de testar código antes de compartilhar
- Monopolizar discussões ou decisões técnicas

## 📚 **Diretrizes Específicas do Projeto**

### **🔧 Git e GitHub:**
- Nunca fazer `git push --force` na main
- Sempre fazer Pull Request para mudanças importantes
- Code review obrigatório para funcionalidades críticas
- Commits devem seguir Conventional Commits
- Issues devem ser claras e bem documentadas

### **💻 Código:**
- Seguir padrões PEP 8 para Python
- Documentar funções complexas
- Não commitar arquivos desnecessários
- Testar funcionalidades antes do push
- Manter consistência com arquitetura escolhida

### **📖 Documentação:**
- README sempre atualizado
- Comentários em código complexo
- Documentar decisões arquiteturais importantes
- Manter guias de instalação funcionais
- Atualizar docs/ quando necessário

### **💬 Comunicação:**
- Respostas no WhatsApp em até 24h (dias úteis)
- Avisar ausências/atrasos no grupo
- Reuniões semanais obrigatórias
- Updates de progresso regulares
- Transparência sobre dificuldades
